## 1. 项目概述

### 1.1 项目背景

精品独立站是一个面向企业级客户的电子商务展示平台，主要功能是展示公司产品和企业信息，客户可以通过询盘方式表达购买意向，取代传统的在线支付功能。

### 1.2 项目目标

- 搭建一个专业、现代的企业展示网站
- 实现商品、新闻、荣誉资质等内容的高效管理
- 提供便捷的询盘系统，替代在线支付
- 确保良好的用户体验和SEO优化
- 前后端分离，支持模块化开发

### 1.3 项目范围

**包含内容：**
- 前台展示网站（5个页面）
- 后台管理系统（8个页面）
- RESTful API 接口
- 数据库设计与实现
- 自动化部署流程

**不包含内容：**
- 复杂的在线支付系统
- 用户注册/登录系统（前台）
- 复杂的订单处理流程

**模块划分原则：**
- 每个文件不超过 800 行代码
- 控制器、服务层分离
- 统一的错误处理
- 清晰的依赖注入

## 4. 数据库设计

### 4.1 数据库选择

**推荐方案：** MySQL 8.0

**选择理由：**
- 与 PHP/Laravel 生态兼容性好
- 部署简单，性能稳定
- 社区支持广泛
- 成本较低

### 4.2 数据表设计

#### 4.2.1 用户表 (`users`)

```sql
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE,
    password VARCHAR(255) NOT NULL,
    email VARCHAR(100) NULL,
    avatar VARCHAR(255) NULL,
    role ENUM('admin', 'editor', 'viewer') DEFAULT 'admin',
    status ENUM('active', 'inactive') DEFAULT 'active',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    last_login_at DATETIME NULL,
    INDEX idx_username (username),
    INDEX idx_role (role)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

#### 4.2.2 商品表 (`products`)

```sql
CREATE TABLE products (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(200) NOT NULL,
    description TEXT NULL,
    price DECIMAL(10, 2) NOT NULL DEFAULT 0.00,
    category VARCHAR(100) NOT NULL,
    sku VARCHAR(50) NULL UNIQUE,
    stock INT DEFAULT 0,
    sales INT DEFAULT 0,
    status ENUM('active', 'inactive', 'out') DEFAULT 'active',
    images JSON NULL,
    specs JSON NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_category (category),
    INDEX idx_status (status),
    INDEX idx_price (price)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

#### 4.2.3 新闻表 (`news`)

```sql
CREATE TABLE news (
    id INT PRIMARY KEY AUTO_INCREMENT,
```

## 5. API接口设计

### 5.1 接口规范

**基础路径：** `/api/v1`

**响应格式：**
```json
{
    "code": 200,
    "message": "success",
    "data": { ... },
    "timestamp": "2024-01-15T10:30:00Z"
}
```

**错误码定义：**
| 状态码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未授权 |
| 403 | 禁止访问 |
| 404 | 资源不存在 |
| 500 | 服务器错误 |

### 5.2 认证接口

| 方法 | 路径 | 描述 |
|------|------|------|
| POST | `/auth/login` | 用户登录 |
| POST | `/auth/logout` | 退出登录 |
| GET | `/auth/profile` | 获取用户信息 |
| PUT | `/auth/password` | 修改密码 |

### 5.3 商品接口

| 方法 | 路径 | 描述 | 权限 |
|------|------|------|------|
| GET | `/products` | 获取商品列表 | 公开 |
| GET | `/products/:id` | 获取商品详情 | 公开 |

```typescript
// modules/product/product.service.ts
@Injectable()
export class ProductService {
  constructor(
    @InjectRepository(Product)
    private productRepository: Repository<Product>,
  ) {}

  async findAll(query: ProductQueryDto) {
    const { page = 1, limit = 10, category, status } = query
    
    const queryBuilder = this.productRepository.createQueryBuilder('product')
    
    if (category) {
      queryBuilder.andWhere('product.category = :category', { category })
    }
    if (status) {
      queryBuilder.andWhere('product.status = :status', { status })
    }
    
    const [items, total] = await queryBuilder
      .orderBy('product.createdAt', 'DESC')
      .skip((page - 1) * limit)
      .take(limit)
      .getManyAndCount()
    
    return {
      items,
      total,
      page,
      limit,
      totalPages: Math.ceil(total / limit),
    }
  }

  async create(createDto: CreateProductDto): Promise<Product> {
    const product = this.productRepository.create(createDto)
    return this.productRepository.save(product)
  }

  async update(id: number, updateDto: UpdateProductDto): Promise<Product> {
    await this.productRepository.update(id, updateDto)
    return this.findOne(id)
  }

  async remove(id: number): Promise<void> {
    await this.productRepository.delete(id)
  }
}
```

- CPU: 2 核
- 内存: 4GB
- 磁盘: 50GB SSD
- 带宽: 5Mbps

**软件环境：**
- Node.js 20.x
- MySQL 8.0
- Nginx
- PM2

**部署步骤：**

1. **安装依赖**
   ```bash
   # 安装 Node.js
   curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
   sudo apt-get install -y nodejs
   
   # 安装 MySQL
   sudo apt-get install mysql-server
   
   # 安装 PM2
   sudo npm install -g pm2
   ```

2. **配置 Nginx**
   ```nginx
   server {
       listen 80;
       server_name api.yourdomain.com;
       
       location / {
           proxy_pass http://localhost:3000;
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection 'upgrade';
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
           proxy_set_header X-Forwarded-Proto $scheme;
           proxy_cache_bypass $http_upgrade;
       }
   }
   ```

3. **配置 PM2**
   ```javascript
   // ecosystem.config.js
   module.exports = {
     apps: [{
       name: 'backend-api',
       script: 'dist/main.js',
       instances: 'max',
       exec_mode: 'cluster',
       env: {
         NODE_ENV: 'production',
         PORT: 3000,
       },
     }],
   }
   ```

4. **部署应用**
   ```bash
   # 克隆代码
   git clone https://github.com/your-repo/backend.git
   cd backend
   
   # 安装依赖
   pnpm install
   
   # 构建
   pnpm build
   
   # 启动 PM2
   pm2 start ecosystem.config.js
   pm2 startup
   pm2 save
   ```

### 8.4 数据库备份

**备份脚本**
```bash
#!/bin/bash
# backup.sh

BACKUP_DIR="/var/backups/mysql"
DATE=$(date +%Y%m%d_%H%M%S)
DB_NAME="ecommerce"
DB_USER="backup"
DB_PASS="your_password"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 执行备份
mysqldump -u$DB_USER -p$DB_PASS $DB_NAME | gzip > $BACKUP_DIR/${DB_NAME}_${DATE}.sql.gz

# 保留最近 7 天的备份
find $BACKUP_DIR -name "*.sql.gz" -mtime +7 -delete

# 上传到云存储（可选）
# aws s3 cp $BACKUP_DIR/${DB_NAME}_${DATE}.sql.gz s3://your-backup-bucket/
```

**定时任务**
```bash
# crontab
0 3 * * * /var/backups/backup.sh
```
